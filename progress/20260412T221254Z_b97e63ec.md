### Key technical observations

- The current `garnir_straightening_step` uses G = {p₁, p₂} where both positions are in the same row. This means swap(p₁, p₂) ∈ RowSubgroup, so `twistedPolytabloid w σ = polytabloid (w * σ)` — making `garnir_twisted_in_lower_span` equivalent to the main theorem (circular for same-row pairs).
- A proper Garnir set should span two rows of a column (positions in the same column but different rows), where the Garnir group permutes elements across rows, genuinely lowering row inversion count.
- The `garnirSet` construction in `PolytabloidBasis.lean` (line 864) does this correctly — it takes two positions in the same column, different rows. But `garnir_straightening_step` currently constructs G from `exists_row_inversion_pair` which gives same-row pairs.
- Fixing this requires restructuring `garnir_straightening_step` to find column-adjacent positions where one row is inverted relative to the other, then using the proper `garnirSet` construction.
