## Current frontier

Two sorry's remain for the Frobenius character formula bridge. The proof strategy:

1. **For N = n**: Use `Theorem5_15_1` which gives `sign(rev) • spechtModuleCharacter = coeff_{toFinsupp + rhoShift}(vandermondePoly * cycleTypePsumProduct)`. Need to match:
   - `shiftedExps` ↔ `toFinsupp + rhoShift` (via `sortedParts_getD_eq_of_antitone`)
   - `alternantDet` ↔ `sign(rev) * vandermondePoly` (via alternant-Vandermonde identity, but circular import blocks direct use)
   - `psumPart` ↔ `cycleTypePsumProduct` for `fullCycleType`

2. **For general N**: `charValue_stability` — charValue is independent of N (standard symmetric function stability). This requires showing that adding zero rows to a bounded partition doesn't change the alternant coefficient.

### Key blockers for the remaining sorry's

- **Circular import**: `FrobeniusCharacterBridge.lean` imports `Theorem5_22_1.lean`, so `alternantDet_eq_sign_mul_vandermondeProd` cannot be used in `Theorem5_22_1.lean`. Would need to either inline it or restructure imports.
- **Polynomial matching**: `psumPart` (MvPolynomial power sum over partitions) vs `cycleTypePsumProduct` (cycle-type based product) — need to show these are equal when evaluated at the same point.
- **Stability**: The general-N case requires a non-trivial stability argument for symmetric function coefficients.
