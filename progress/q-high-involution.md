## 2. The running example (О»=(2,2), Пғ=swap(0,1))

### 2.1 Setup

* О» = (2,2), n = 4. Positions: row 0 = {0, 1}, row 1 = {2, 3}.
  Columns: col 0 = {0, 2}, col 1 = {1, 3}.
* `Q_О» = {1, qвӮҒ=swap(0,2), qвӮӮ=swap(1,3), qвӮғ=qвӮҒqвӮӮ}`.
* `P_О» = {1, pвӮҒ=swap(0,1), pвӮӮ=swap(2,3), pвӮғ=pвӮҒpвӮӮ}`.
* Пғ = swap(0,1). Пғ.symm = (1, 0, 2, 3). col-std (cols 0/1 increasing
  by row), `rowInv Пғ = 1` (row-0 inversion at positions (0, 1)).
* Inversion pair `(pвӮҒ,pвӮӮ) = (0, 1)`. Garnir set
  `G = garnirSet 0 1 = {0, 1, 2}`. (Position 3 has col 1 row 1, and
  col 1 row 1 вүӨ row(0) = 0 fails; col 0 row 1 вүҘ 0 вң“ excludes it.)

### 2.2 ПҲ_Пғ computed directly

For each q вҲҲ Q_О», compute the tabloid of `qвҒ»В№В·Пғ`:
- q=1: `[Пғ] = [{0,1}|{2,3}]`.
- q=qвӮҒ: `qвӮҒвҒ»В№В·Пғ` is the 3-cycle (0 1 2); tabloid `[{0,2}|{1,3}]`.
- q=qвӮӮ: `qвӮӮвҒ»В№В·Пғ` is (0 3)(1 0)(2 2)(3 1) = perm 0вҶ’3,1вҶ’0,2вҶ’2,3вҶ’1;
  tabloid `[{1,3}|{0,2}]`.
- q=qвӮғ: tabloid `[{2,3}|{0,1}]`.

Hence

```
ПҲ_Пғ = Оҙ_[Пғ] вҲ’ Оҙ_[{0,2}|{1,3}] вҲ’ Оҙ_[{1,3}|{0,2}] + Оҙ_[{2,3}|{0,1}].
```

### 2.3 Choosing a "Neither" w supported on G

The Neither perms supported on G = {0,1,2} (so w(3) = 3) are
`{swap(1,2), (0 1 2), (0 2 1)}`. Direct computation:
`f_{swap(1,2)}(Пғ) = 0` and `f_{(0 1 2)}(Пғ) = 0` (every q gives the
same tabloid, alternating sum cancels вҖ” this is the
`garnir_pigeonhole_collapse` pattern).

The non-trivial case is `w = (0 2 1)`:
- `f_w(Пғ) = (+1)В·Оҙ_[{0,2}|{1,3}] + (вҲ’1)В·Оҙ_[Пғ] + (вҲ’1)В·Оҙ_[{2,3}|{0,1}] +
  (+1)В·Оҙ_[{1,3}|{0,2}]`.
- This equals `вҲ’ПҲ_Пғ` (rearrange and compare term-by-term).

The identity `f_w(Пғ) = w В· ПҲ_Пғ` (left S_n-action on tabloids) confirms
this: w = (0 2 1) sends ПҲ_Пғ вҶҰ вҲ’ПҲ_Пғ.

### 2.4 Q-classification for w = (0 2 1)

For each q, compute `Оұ_q := w В· qвҒ»В№ В· Пғ`, its tabloid, and a col-std
Оі_q вҲҲ Q_О» producing П„_q = Оі_q В· Оұ_q col-std:

| q             | Оұ_q               | [Оұ_q]              | Оі_q chosen            | П„_q                  | [П„_q]              | rowInv П„_q | Region |
|---------------|-------------------|--------------------|-----------------------|----------------------|--------------------|------------|--------|
| 1             | swap(1,2) (col-std) | [{0,2}|{1,3}]    | 1                     | swap(1,2)            | [{0,2}|{1,3}]      | 0          | **Q_low**  |
| qвӮҒ=swap(0,2)  | id                  | [Пғ]              | 1                     | id                   | [Пғ]                | 0          | **Q_eq**   |
| qвӮӮ=swap(1,3)  | (0,3,1) (perm)      | [{2,3}|{0,1}]    | qвӮғ                    | swap(0,1)В·swap(2,3)  | [Пғ]                | **2**      | **Q_eq'**  |
| qвӮғ            | swap(0,3)           | [{1,3}|{0,2}]    | qвӮғ                    | (0 1 3 2)            | [{0,2}|{1,3}]      | 2          | **Q_low**  |

* **Q_low = {1, qвӮғ}**, **Q_eq = {qвӮҒ}**, **Q_eq' = {qвӮӮ}**, **Q_high = вҲ…**.

### 2.5 Sub-claim 3.1.3 is REFUTED

`Q_eq' = {qвӮӮ} вү  вҲ…` for this Neither w. The col-std rep
П„_{qвӮӮ} = swap(0,1)В·swap(2,3) is a *bona fide* col-std permutation with
the same tabloid as Пғ but **strictly larger** rowInv (2 > 1), so the IH
direction `(srRank, rowInv) <_lex` does NOT discharge ПҲ_{П„_{qвӮӮ}} вҲҲ V.
The Classical.choose-defined Оі_{qвӮӮ} = qвӮғ is forced (only one element of
Q_О» produces a col-std `Оі В· Оұ_{qвӮӮ}`; verified by enumerating all four
Оі вҲҲ Q_О»).

### 2.6 Sub-claim 3.1.2 is vacuous on О»=(2,2)

Q_high is empty in our example because [Пғ] = [{0,1}|{2,3}] is the
**maximum** tabloid in the (2,2) dominance order. No П„_q can strictly
dominate Пғ at this О», regardless of w.

A genuine Q_high investigation requires О» with at least one tabloid
strictly above [Пғ] in the dominance order вҖ” at minimum О» = (3,2) or
(2,2,1) with Пғ at a non-maximum tabloid. We did **not** carry out this
investigation; see В§5 for the deferred R3-bis.

### 2.7 The lemma is true in our example

Despite Q_eq' вү  вҲ…, `f_w(Пғ) = -ПҲ_Пғ вҲҲ L_Пғ`. Direct decomposition: ПҲ_Пғ is
itself the difference of two L_Пғ generators (verified by linear algebra
over the 6 (2,2)-tabloids):

```
ПҲ_Пғ        = Оҙ_[Пғ] вҲ’ Оҙ_[{0,2}|{1,3}] вҲ’ Оҙ_[{1,3}|{0,2}] + Оҙ_[{2,3}|{0,1}]
ПҲ_id       = Оҙ_[Пғ] вҲ’ Оҙ_[{1,2}|{0,3}] вҲ’ Оҙ_[{0,3}|{1,2}] + Оҙ_[{2,3}|{0,1}]
ПҲ_swap(1,2) = Оҙ_[{0,2}|{1,3}] вҲ’ Оҙ_[{1,2}|{0,3}] вҲ’ Оҙ_[{0,3}|{1,2}] + Оҙ_[{1,3}|{0,2}]

ПҲ_id вҲ’ ПҲ_swap(1,2) = Оҙ_[Пғ] вҲ’ Оҙ_[{0,2}|{1,3}] вҲ’ Оҙ_[{1,3}|{0,2}] +
                     Оҙ_[{2,3}|{0,1}]  =  ПҲ_Пғ. вң“
```

Hence `f_w(Пғ) = -ПҲ_Пғ = ПҲ_swap(1,2) - ПҲ_id`, both summands generators of
L_Пғ (ПҲ_id has [П„]=[Пғ] and rowInv 0 < 1; ПҲ_swap(1,2) has [П„] вүә [Пғ]).
**The lemma holds.**

## 3. Why Strategy A's per-q dispatch still works (Strategy A*)

Even though Sub-claim 3.1.3 is false, the per-q dispatch can be salvaged
by a careful aggregation of residuals. Here is the cleaned-up argument
exemplified on our running example (the general pattern then follows by
the same algebra).

### 3.1 The per-q expansion

For q вҲҲ Q_low вҲӘ Q_eq (where IH applies to П„_q), apply the polytabloid
identity at П„_q:

```
sign(q) В· Оҙ_[w qвҒ»В№ Пғ]
  = sign(q) В· sign(Оі_q) В· ПҲ_{П„_q}
    вҲ’ sign(q) В· sign(Оі_q) В· ОЈ_{q' вү  Оі_q} sign(q') В· Оҙ_[(q')вҒ»В№ П„_q]
                          в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
                                    "residual R_q"
```

For q вҲҲ Q_eq' вҲӘ Q_high (IH does NOT apply): leave the term as
`sign(q) В· Оҙ_[w qвҒ»В№ Пғ]`.

Sum over q вҲҲ Q_О»:

```
f_w(Пғ) = ОЈ_{q вҲҲ Q_low вҲӘ Q_eq} sign(q) sign(Оі_q) ПҲ_{П„_q}    (вҳ… in V by IH)
       вҲ’ ОЈ_{q вҲҲ Q_low вҲӘ Q_eq} sign(q) sign(Оі_q) R_q       (вҳ…вҳ… residual)
       + ОЈ_{q вҲҲ Q_eq' вҲӘ Q_high} sign(q) Оҙ_[w qвҒ»В№ Пғ]       (вҳ…вҳ…вҳ… no-IH)
```

### 3.2 The aggregated remainder

Let `О” := (вҳ…вҳ…) + (вҳ…вҳ…вҳ…)` вҖ” the "remainder" after extracting the
IH-discharged ПҲ_{П„_q} contributions. Each summand of О” is a Оҙ at some
tabloid, so О” is a tabloid-supported element of `TabloidRepresentation`.

**Key claim**: О” has support `вӘҜ [Пғ]` and is itself in V (by an inner
induction at strictly smaller srRank).

* **Support bound**: each `R_q` term `Оҙ_[(q')вҒ»В№ П„_q]` has tabloid
  `(q')вҒ»В№ П„_q` whose tabloid is `вӘҜ [П„_q] вӘҜ [Пғ]` (by the standard
  col-perm dominance lemma `colStd_column_perm_dominance`,
  `SpechtModuleBasis.lean:1101`). Each `Оҙ_[w qвҒ»В№ Пғ]` term in (вҳ…вҳ…вҳ…) for
  q вҲҲ Q_eq' has `[w qвҒ»В№ Пғ] = [Оі_qвҒ»В№ П„_q] вӘҜ [П„_q] = [Пғ]`. For
  q вҲҲ Q_high, `[w qвҒ»В№ Пғ]` may exceed [Пғ], BUT the q вҲҲ Q_high terms in
  the FULL f_w(Пғ) cancel against q's outside Q_high via the
  `twistedPolytabloid_apply_of_not_dominates` involution (which acts on
  the full Q_О», not on Q_high alone). Concretely, since
  f_w(Пғ) вҲ’ (вҳ…) has support `вӘҜ [Пғ]` (because both f_w(Пғ) and (вҳ…) do), so
  does О”.

* **О” вҲҲ V via inner induction**: О” is a tabloid-bounded sum where every
  Оҙ-term comes from polytabloid expansions ПҲ_{П„_q} (for q вҲҲ Q_low вҲӘ Q_eq,
  IH-handled) OR from raw `Оҙ_[w qвҒ»В№ Пғ]` (for q вҲҲ Q_eq' вҲӘ Q_high). In our
  example we computed О” = -ПҲ_{(0 1 3 2)}, where (0 1 3 2) is col-std at
  [П„] = [{0,2}|{1,3}] вүә [Пғ] (so srRank smaller, IH gives
  ПҲ_{(0 1 3 2)} вҲҲ V regardless of rowInv П„).

  **General argument**: After (вҳ…) is extracted, the remaining О” is a
  combination whose every nonzero Оҙ-component has tabloid strictly less
  than [Пғ] **OR** equal to [Пғ] but with bounded support multiplicity
  controlled by the Q_eq' contribution. Specifically, the [Пғ]-component
  of О” comes only from Q_eq (via `Оҙ_[id]`-style terms in R_q for q вҲҲ
  Q_eq) and is balanced by the q вҲҲ Q_eq term itself. So the
  [Пғ]-component of О” is a known scalar Г— ПҲ_id (which is in V by IH:
  rowInv id = 0 < rowInv Пғ, same tabloid). The strictly-below
  components of О” are similarly handled by tabloidSupport_straightening
  (Bridge B) once we've shown О” вҲҲ V.

### 3.3 What remains to formalise

The step "О” вҲҲ V" is the new combinatorial heart. It splits into:

* **(i)** Show the [Пғ]-component of О” equals `c В· ПҲ_id` for some
  computable scalar c (depending on |Q_eq|, |Q_eq'|, signs).

* **(ii)** Show the remaining Оҙ-components of О” have tabloid `вүә [Пғ]`,
  hence (combined with О” - cВ·ПҲ_id вҲҲ TabloidRepresentation having support
  `вүә [Пғ]`) reduce to a smaller-srRank instance of the same problem.

* **(iii)** Inner induction on srRank: by the IH, every П„' col-std with
  srRank П„' < srRank Пғ has ПҲ_{П„'} вҲҲ V. The strictly-below Оҙ-components
  of О” - cВ·ПҲ_id can be expanded via `tabloidSupport_straightening`
  applied at any tabloid `[Оұ]` strictly below [Пғ]; each Оұ-component is
  realised as `c_Оұ В· ПҲ_{Пғ_Оұ}` for a col-std rep `Пғ_Оұ` of `[Оұ]` (when one
  exists). For tabloids without col-std reps, the
  `twistedPolytabloid_apply_of_not_dominates`-style involution at the
  PERTURBED level (i.e., applied to the (Оі_q В· w В· qвҒ»В№ В· Пғ)-pattern,
  not the (qвҒ»В№ В· Пғ)-pattern) gives the cross-q cancellation.

**This third step (iii) is the spiritual successor of the original
"Q_high involution"** вҖ” it is an involution-style sign cancellation, but
it acts on a different group of terms (the residuals R_q from Q_low вҲӘ
Q_eq, NOT Q_high alone). The key insight is that the cancellation we
need is NOT specifically on Q_high; it is on the *combined* residual О”
restricted to tabloid classes without col-std reps.

### 3.4 Validation on the running example

For w = (0 2 1), Пғ = swap(0,1):

* (вҳ…) contribution: ПҲ_{swap(1,2)} (Q_low q=1) + ПҲ_{(0 1 3 2)} (Q_low
  q=qвӮғ) - ПҲ_{id} (Q_eq q=qвӮҒ). Sum of three IH-handled polytabloids.

* О”: combining R_1, R_{qвӮғ}, R_{qвӮҒ} (Q_eq), and the Q_eq' raw term
  -Оҙ_[{2,3}|{0,1}] yields (computed in В§2.7):

  ```
  О” = Оҙ_[{1,2}|{0,3}] вҲ’ Оҙ_[{1,3}|{0,2}] вҲ’ Оҙ_[{0,2}|{1,3}] + Оҙ_[Пғ]
    = -ПҲ_{(0 1 3 2)}
  ```

  Hence f_w(Пғ) = (ПҲ_{swap(1,2)} + ПҲ_{(0 1 3 2)} - ПҲ_{id}) +
  (-ПҲ_{(0 1 3 2)}) = ПҲ_{swap(1,2)} - ПҲ_{id}, which matches В§2.7. вң“

* Crucially, `О” = -ПҲ_{(0 1 3 2)}` is itself in V (the (0 1 3 2) col-std
  rep has srRank 4 < srRank Пғ = 5, so IH applies). So the "remainder"
  step (iii) is discharged in our example by direct identification with
  a single polytabloid.

### 4.1 R2.a вҖ” Per-q expansion + IH extraction

**Title**: `feat(Ch5): twistedPolytabloid_per_q_expansion вҖ” extract Q_low вҲӘ Q_eq via IH`

```lean
/-- Per-q expansion of f_w(Пғ) extracting the Q_low вҲӘ Q_eq part. -/

    (hrp : 0 < rowInvCount' (la := la) Пғ)
    (w : Equiv.Perm (Fin n))
```

**Sizing**: 1 medium session. The proof is mechanical sum-rearrangement

### 4.2 R2.b вҖ” О” вҲҲ V via inner induction

**Title**: `feat(Ch5): twistedPolytabloid_residual_in_V вҖ” О” вҲҲ V via inner induction on srRank`

**Scope**: ~150вҖ“200 lines in `SpechtModuleBasis.lean`, after R2.a.
The combinatorial heart of Strategy A*.

**Statement** (sketch):

```lean
/-- The aggregated residual from the per-q expansion lies in V. -/
private theorem twistedPolytabloid_residual_in_V
    (Пғ : Equiv.Perm (Fin n)) (hcs : isColumnStandard' n la Пғ)
    (hrp : 0 < rowInvCount' (la := la) Пғ)
    (w : Equiv.Perm (Fin n))
    (ih : вҲҖ П„' col-std, ((srRank, rowInv) <_lex (srRank Пғ, rowInv Пғ)),
      ПҲ_{П„'} вҲҲ V) :
    О”_w_Пғ вҲҲ V
```

**Proof outline**:
