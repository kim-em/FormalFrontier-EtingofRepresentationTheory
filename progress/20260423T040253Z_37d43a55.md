## Accomplished
- Claimed #2380 (garnir_twisted_in_lower_span) and analyzed the theorem structure.
- **Discovered a circular-dependency bug in the theorem statement**: for
  `w ∈ RowSubgroup` (e.g. `swap(p₁, p₂)`), the twisted polytabloid reduces to
  `f_w(σ) = ψ_σ` exactly (because w preserves row assignments on positions, so
  `toTabloid(w·q⁻¹·σ) = toTabloid(q⁻¹·σ)`). The current hypothesis
  `w ∉ ColumnSubgroup` (added by PR #2412) does not exclude such w, so the
  theorem claims `ψ_σ ∈ L` — true by the straightening theorem, but the
  Garnir step is precisely what proves the straightening theorem. Circular.
- Verified the issue with concrete computation on λ = (2,1), σ = swap(0,1),
  w = swap(p₁, p₂): `f_w(σ)` is literally `ψ_σ`.
- Also identified a distinct obstacle: the **k = 0 case** (both
  `col(p₁)`, `col(p₂)` length 1) where the Garnir identity is the tautology
  0 = 0 and gives no reduction. In this case `swap(p₁, p₂)·σ` is still
  column-standard with strictly fewer row inversions, and `ψ_σ = ψ_{swap·σ}`
  because `swap(p₁,p₂)` commutes with every q ∈ Q (single-cell columns, so q
  fixes p₁ and p₂). This means the k=0 case should be handled via a direct
  swap reduction, NOT via Garnir.
- Filed **#2413** with the proposed architectural fix: add
  `hw_row : w ∉ RowSubgroup n la` hypothesis, split `garnir_straightening_step`
  into k=0 (swap reduction) and k≥1 (three-way sum split: Q, P\Q, neither)
  cases. Listed six helper lemmas the fix will need.
