**Core finding.** The plan's Step 1 (`TP(w, σ) ∈ V^λ` interpreted
as `TP ∈ Submodule.span ℂ (Set.range polytabloidTab)`) cannot be
discharged upstream of `garnir_twisted_in_lower_span` (line 1600 in
`SpechtModuleBasis.lean`). The plan's suggested approach
(`apply Submodule.smul_mem` based on `TP = w · ψ_σ`) doesn't work:

1. The natural `S_n`-action on `TabloidRepresentation` (induced from
   `V_λ` via `tabloidProjection`) is RIGHT translation by `w⁻¹`. Under
   this action, `w • gPT σ = ∑_q sign(q) δ_{[q⁻¹ σ w⁻¹]}` ≠ `TP =
   ∑_q sign(q) δ_{[w q⁻¹ σ]}`.
2. "Left translation by `w`" is NOT a well-defined linear map on
   tabloid classes (it would require `w P_λ w⁻¹ = P_λ`, generally
   false).
3. Direct: `TP = (1/|P_λ|) tabloidProjection(of(σ⁻¹) · ColAntisym ·
   of(w⁻¹) · RowSym)`, but the inner element is NOT in `V_λ` and
   showing it's in `V_λ + ker(tabloidProjection)` is itself a
   non-trivial sub-lemma.
4. Bypassing `tabloidSupport_straightening` via direct Algorithm A
   (col-std reps): col-std reps don't always exist (e.g.,
   `[{2,3}|{0,1}]` for λ=(2,2) has no col-std rep, but it appears
   in supp(TP) for the Case I counter-example).
5. Algorithm A via SYT reps: requires every dominance-maximal element
   of `supp(v)` (at each step) to be an SYT-tabloid. This holds
   classically because `v ∈ span polytabloidTab T` — but proving
   *that* upstream is the original issue.

The classical math IS correct (Case I verified: `TP = -e_{T_1} +
e_{T_2}`), but the *proof* is blocked.
