### Key technical discoveries

- `set p := 1` causes catastrophic variable renaming: since `1` appears in `Fin (k + 1)` types, `set` replaces it with `p`, renaming `u`, `v`, `adj` to `u✝`, `v✝`, `adj✝`. Fix: don't use `set` for values that appear in types.
- For D_n with variable `k`: `simp [DynkinType.adj]` times out because the ite chain is too complex. Fix: use `show` to state the explicit ite form after unfolding, then `split_ifs <;> omega`.
- For E-types with concrete `k`: `dsimp only [DynkinType.adj]` (not `simp only`) properly reduces the match expression, then `split_ifs <;> omega` closes goals.
- `simp only` vs `dsimp only`: `simp only [DynkinType.adj]` tries simplification rules which can interact badly with the context; `dsimp only [DynkinType.adj]` does pure definitional unfolding which is faster and more predictable.
