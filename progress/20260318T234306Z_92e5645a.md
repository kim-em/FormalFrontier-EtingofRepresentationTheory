### Not completed

The `equivAt_eq_sink` proof (first isomorphism theorem at vertex i) remains sorry'd.

**Root cause:** After unfolding `reflectionFunctorMinus` at vertex i, the goal becomes:
```
((⊕_{a} F⁺(V).obj a.fst) ⧸ range(sourceMap)) ≃ₗ[k] ρ.obj i
```
Constructing a `LinearMap` from this direct sum to `ρ.obj i` fails because:
1. `DirectSum.toModule` needs `AddCommMonoid` and `Module` instances for each component
2. The component types are `Decidable.rec ...` which Lean cannot synthesize instances for
3. Using `generalize` + `cases` to reduce Decidable.rec doesn't work because the pattern
   `inst a.fst i` is no longer present after unfolding

**Potential solutions (for future agents):**
- Option A: Refactor `reflectionFunctorPlus`/`Minus` definitions to use `dite` instead of `Decidable.casesOn` — this would let `simp` reduce types
- Option B: Build forward/backward linear maps for the `LinearEquiv` manually using `Subtype.val` and `Submodule.Quotient.mk`, bypassing `DirectSum.toModule`
- Option C: Construct an `Equiv` between the two direct sum index types first, then use `DirectSum.lequivCongrLeft` + `congrLinearEquiv`
