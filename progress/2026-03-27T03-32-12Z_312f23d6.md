## Accomplished

Worked on issue #1750 "Prove dominance order properties for tabloid column permutations".

Added 4 fully proved theorems and 1 theorem with 1 remaining sorry to `Chapter5/TabloidModule.lean`:

1. **`syt_entry_lt_of_row_lt`** (proved): In a standard tableau, entries in the same column are ordered by row — if row(T(k1)) < row(T(k2)) in the same column, then k1 < k2.

2. **`syt_col_entry_le_of_row_le`** (proved): The `≤` version of the above, via `eq_or_lt` case split.

3. **`swap_column_dominance`** (proved): For a single column transposition swapping positions p1, p2 in the same column with row(p1) < row(p2), the tabloid `{swap(p1,p2) * sigma}` is dominated by `{sigma}`. Uses a 3-case analysis on row positions relative to threshold i, with an injection argument via `Equiv.swap e1 e2` for the critical middle case.

4. **`column_perm_dominance`** (1 sorry): General column permutation dominance — for any q in Q_lambda, `{q^{-1} * sigma_T}` is dominated by `{sigma_T}`. Extending swap_column_dominance to general permutations requires a per-column Finset decomposition argument.

5. **`column_perm_strict_dominance`** (proved): Strict dominance follows from dominance + the existing `ColumnSubgroup_ne_tabloid` result for non-identity q.

## Current frontier

The remaining sorry in `column_perm_dominance` requires proving that for each column c, the count of entries ≤ k in rows 0..i-1 equals min(t_c, s_c) where t_c = cells with row < i and s_c = entries ≤ k. This is ~50-100 lines of Finset infrastructure. An alternative approach using `Equiv.Perm.swap_induction_on` was considered but doesn't work because intermediate permutations may lose the column-preserving property.
