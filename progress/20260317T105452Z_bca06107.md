## Accomplished

Completed issue #863 — made `Etingof.reflectionFunctorPlus` (Definition 6.6.3) sorry-free.

The core challenge was dependent type engineering: the object function branches on `Decidable (v = i)` at the type level, and the `AddCommMonoid`/`Module` instances and `mapLinear` field must all branch consistently. The solution uses:

- A shared `let dec` Decidable instance across all fields
- A `branch` helper that parameterizes the type-level branching by the Decidable value
- `@Decidable.rec` with explicit motives for `instAddCommMonoid` and `instModule`, ensuring types reduce in lockstep
- `dsimp only [branch]` in the mapLinear proof to expose the Decidable.rec for match reduction
- Four-case dispatch on `(dec a, dec b)` for mapLinear: two vacuous cases (exfalso), one reversed arrow (DirectSum.component composed with ker subtype), one unchanged arrow
