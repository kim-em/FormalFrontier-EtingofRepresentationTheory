### Definition 9.3.1 — cartanMatrix
- **Mathematical accuracy**: Correct. The Cartan matrix C = (c_{ij}) where c_{ij} = [P_j : M_i] is the Jordan-Hölder multiplicity. The formalization takes `jhMultiplicity` as a parameter and wraps it with `Matrix.of`, which is correct given missing Mathlib infrastructure.
- **Note**: The definition is essentially trivial (`Matrix.of jhMultiplicity`). The real mathematical content is deferred to the jhMultiplicity parameter. This is an acceptable parametric approach.
- **Name collision**: `Etingof.cartanMatrix` is also defined in Chapter 6 (Definition 6.4.1) for a completely different concept (graph Cartan matrix A = 2·Id - R). The type signatures differ so Lean can disambiguate, but this could cause confusion. Not blocking since they're in separate chapter imports.
- **Minor issue**: Name collision worth tracking for future cleanup.

### Definition 9.7.1 — MoritaEquivalent
- **Mathematical accuracy**: Correct modulo a documented deviation. The book defines Morita equivalence via finite-dimensional module categories (A-fmod ≌ B-fmod). The formalization uses full module categories (ModuleCat A ≌ ModuleCat B) because Mathlib lacks a standalone finite-dimensional module category. Docstring explains this.
- **Universe handling**: `max u v` is correct for ensuring equivalence lives in a common universe.
- **Naming note**: Could arguably be `IsMoritaEquivalent` to match the `Is` prefix convention for predicates, but "Morita equivalent" reads naturally as is. Not blocking.
- **No blocking issues.**
