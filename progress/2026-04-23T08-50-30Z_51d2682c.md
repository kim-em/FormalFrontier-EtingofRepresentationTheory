## Accomplished

Refactored `garnir_twisted_in_lower_span`, `garnir_straightening_step`, and
`polytabloidTab_column_standard_in_span` in
`EtingofRepresentationTheory/Chapter5/SpechtModuleBasis.lean` to use the
tabloid-strict-dominance order instead of `rowInvCount'` as the Garnir
straightening induction measure. Closes #2424.

Key changes:

- Added `srRank la σ` (count of permutations whose tabloid strictly
  dominates σ's tabloid) with two helper lemmas:
  - `srRank_eq_of_toTabloid_eq`: invariance under tabloid equivalence.
  - `srRank_lt_of_tabloidStrictDominates`: strict decrease under strict
    dominance.
- `garnir_twisted_in_lower_span` signature: conclusion now says
  `twistedPolytabloid w σ ∈ span {ψ_τ : τ col-std, tabloidStrictDominates la τ σ}`
  (was `rowInvCount'(τ) < rowInvCount'(σ)`). This is the correct classical
  Garnir statement. Body remains `sorry` (same sorry, restated).
- `garnir_straightening_step` signature and internal proof updated to
  conclude with a **disjunction**:
    `tabloidStrictDominates la τ σ ∨
       (toTabloid τ = toTabloid σ ∧ rowInvCount'(τ) < rowInvCount'(σ))`.
  The first disjunct covers the main case (k ≥ 1) via
  `garnir_twisted_in_lower_span` with an injection `h_strict_sub_L`. The
  second covers the k = 0 corner case: σ' = t·σ (row swap) has the same
  tabloid as σ and strictly fewer row inversions.
- `polytabloidTab_column_standard_in_span` outer proof uses **nested
  strong induction**: outer on `srRank` (for the strict-dominance
  branch), inner on `rowInvCount'` at fixed `srRank` (for the same-
  tabloid branch).

Ups:

- The issue author's suggestion that "a single swap of a row-inversion pair
  produces a strictly dominating tabloid" is incorrect — a row swap keeps
  the tabloid unchanged. The actual refactor uses the lex measure described
  above to handle the k=0 case cleanly, with the row-swap case contributing
  via `rowInvCount'` strict decrease within a fixed tabloid class.

Sorry count unchanged: 39 before → 39 after. Only sorry in SpechtModuleBasis
is still `garnir_twisted_in_lower_span` (restated with new signature).
