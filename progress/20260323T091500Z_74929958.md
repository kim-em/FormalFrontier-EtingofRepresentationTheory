### Proof structure (4 phases)

1. **Phase 1**: Expand |S(g)|² = (∑_x f)(∑_y conj(f)) via `Fintype.sum_mul_sum`, swap sums with `Finset.sum_comm`
2. **Phase 2**: For each x, reindex g → x⁻¹gx via `MulAut.conj x⁻¹` and y → x⁻¹y via `Equiv.mulLeft x⁻¹`. Key identity: `(x⁻¹y)⁻¹(x⁻¹gx)(x⁻¹y) = y⁻¹gy` (by `group`)
3. **Phase 3**: Body is x-independent → factor out |GL₂| via `Finset.sum_const` + `nsmul_eq_mul`
4. **Phase 4**: Convert GL₂-filtered sum (with dite on K-membership) to K-subtype sum via `Finset.sum_congr_set`

### Key technical discoveries

- `simp_rw` handles rewrites inside `dite` dependent types (e.g., `simp_rw [h_grp]` where h_grp rewrites the conjugation expression inside dite proof terms)
- `Finset.sum_congr_set` is the right tool for converting between Fintype sums over GL₂ with dite-based body functions and subtype sums over K
- When `sum_congr_set` provides `hk : k ∈ S.carrier`, you need `have hk' : k ∈ S := hk` before `split_ifs` can resolve `dite(k ∈ S, ...)` — the `.carrier` form doesn't match syntactically
