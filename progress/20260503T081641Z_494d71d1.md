**Mathematical finding (full text on the issue comment):** The
issue's proposed proof of "col-std existence sub-lemma for
`[α] = [σ]` case" (column-restandardize `w q⁻¹ σ` to get a col-std
`τ_α` with `[τ_α] = [σ]` and `rowInv τ_α < rowInv σ`) is unsound:

- From `[w q⁻¹ σ] = [σ]` we get `w q⁻¹ ∈ P_λ`. Set `p := w q⁻¹`.
- Column-restandardization yields `γ ∈ Q_λ` with `γ · p · σ` col-std,
  but `[γ · p · σ] = [σ]` requires `γ ∈ P_λ`, hence `γ = 1` (since
  `P ∩ Q = {1}`). So the construction only gives `[τ_α] = [σ]` when
  `p · σ` is **already** col-std, which the assumptions `w ∉ Q` and
  `q ∈ Q` prevent generically.
- The simpler swap-based variant (apply `t = swap(p₁,p₂) ∈ P_λ` to
  σ for a row inversion `(p₁,p₂)`) also fails: `t · σ` is **not**
  generally col-std, because for pairs `(q₁, p₁)` in the same column
  with `q₁` above `p₁`, col-std at this pair would require
  `(t·σ).symm(q₁) = σ.symm(q₁) < σ.symm(p₂) = (t·σ).symm(p₁)`, which
  is not implied by `σ` being col-std and `(p₁,p₂)` being a row
  inversion.
- The existing `garnir_straightening_step` k=0 corner case
  (`SpechtModuleBasis.lean:2080-2255`) makes the analogous swap
  work, but only because `k=0` forces col(p₁) and col(p₂) to be
  **singleton columns** — a much stricter setting than the aux
  helper's general assumptions.

**Why the IH case (b) is essential:** the λ=(2,2) tabloid
`[{2,3}|{0,1}]` admits no col-std rep at all (top-of-column entries
{2,3} ⊆ {2,3} cannot be < bottom-of-column entries {0,1}), so
`[α] ≺ [σ]` cases also can fail to admit a col-std rep. The IH-case
dispatch is therefore not a fallback but the dominant work — and the
issue's outline does not specify it concretely enough to follow.
