### Key techniques discovered

1. **`subst hv` → `rw [hv]`**: After `by_cases hv : v = i`, using `subst hv` eliminates `i` from the context (Lean substitutes the more recently introduced variable), causing "Unknown identifier `i`" in downstream code. Using `rw [hv]` rewrites the goal without eliminating variables.

2. **Fintype instance mismatch in dim vector bridge**: `simpleReflectionDimVector_eq_simpleReflection` constructs its own Fintype instance internally via `haveI`, while the goal uses a different instance (`hFT_into`/`hFT_out`). These are propositionally but not definitionally equal, so `rw` fails. Solution: prove the dim equality via `simpleReflection` (no Fintype dependency), then `convert ... using 2` to bridge back to `simpleReflectionDimVector`.

3. **`let fm` for F⁻**: `reflectionFunctorMinus` takes `[Fintype (ArrowsOutOf V i)]` as an instance parameter, unlike `reflectionFunctorPlus` which does not. Using `let fm := @Etingof.reflectionFunctorMinus ... hFT_out` avoids repeating the Fintype argument everywhere.

4. **`Equiv.subsingleton_congr` explicit**: The dot notation `.toEquiv.subsingleton_congr` fails across line breaks and in certain contexts after `rw`. Using `Equiv.subsingleton_congr (e.toEquiv)` explicitly avoids these issues.
