  brute-force unfold+rfl proofs for the naturality conditions.

- Attempted approaches for ne_ne naturality:
  - Massive unfold + `rw [h_da, h_db]`: fails with "motive is not type correct" (dependent types)
  - `simp only [h_da, h_db]`: h_da/h_db unused (simp can't rewrite inside match terms)
  - `simp only [dif_neg ha, dif_neg hb]`: successfully reduces by_cases in equivAt lambda
  - `exact match inst_dec a i, inst_dec b i with ...`: fails with "Application type mismatch"
    because ▸ transport doesn't reduce inside match branches

### Recommended approach for remaining sorries

The correct approach for the naturality proofs is likely:
1. Express each operation (reversedArrow_ne_ne, ▸ transport) as a `cast` with explicit type equalities
2. Use `cast_cast` to compose the casts
3. Show the composite is `cast h e` where `h : (a ⟶ b) = (a ⟶ b)`, then `cases h; rfl`

Alternatively, restructure `reversedAtVertex_twice` to avoid `Quiver.ext'`/`funext`, using
a more computation-friendly proof (e.g., direct `Quiver.mk` construction with `rfl` components).
