## Current frontier

`column_standard_in_span'` needs a new proof technique. With `σ = p * sytPerm(T)`, we get `of(σ) * c_λ = of(p) * polytabloid(T)`. Need to show `of(p) * polytabloid(T) ∈ span(polytabloids)` for p ∈ P_λ. Possible approaches:
1. Show P_λ action preserves the span of standard polytabloids (Specht module is a P_λ-representation)
2. Use the straightening lemma recursively on `of(p) * polytabloid(T)`
3. Show `of(p) * polytabloid(T) = polytabloid(T)` directly (this would require `of(p) * of(sytPerm T) * c_λ = of(sytPerm T) * c_λ`, i.e., RIGHT absorption `of(p) * a_λ = a_λ` applied to the full `of(sytPerm T) * a_λ * b_λ` — this actually DOES work since `of(p) * a_λ = a_λ`)

Wait — approach 3 is actually correct! `of(p) * of(sytPerm T) * a_λ * b_λ = of(sytPerm T) * (of(p') * a_λ) * b_λ` where `p' = sytPerm(T)⁻¹ * p * sytPerm(T)`. But `p'` is a conjugate, not necessarily in P_λ. So this doesn't work directly.

Actually: `of(p) * polytabloid(T) = of(p) * of(sytPerm T) * c_λ = of(p * sytPerm T) * c_λ = of(σ) * c_λ`. And we want `of(σ) * c_λ ∈ span(polytabloids)`. This is circular — it's the very thing we're trying to prove.

The real fix may be to use RIGHT absorption differently: `of(σ) * c_λ = of(σ) * a_λ * b_λ = of(σ) * of(p⁻¹) * a_λ * b_λ = of(σ * p⁻¹) * c_λ = of(sytPerm T) * c_λ = polytabloid(T)`. This uses `a_λ * of(p⁻¹) = a_λ` (RIGHT absorption). But wait, `c_λ = a_λ * b_λ` so `of(σ) * c_λ = of(σ) * a_λ * b_λ`. We need `of(σ) * a_λ = of(σ * p⁻¹) * of(p) * a_λ = of(sytPerm T) * of(p) * a_λ = of(sytPerm T) * a_λ` (using `of(p) * a_λ = a_λ`). YES! This works because `σ = p * sytPerm(T)` means `of(σ) = of(p) * of(sytPerm T)`, so:
```
of(σ) * a_λ * b_λ = of(p) * of(sytPerm T) * a_λ * b_λ
```
And LEFT absorption gives `of(p) * a_λ = a_λ` for p ∈ P_λ. But `of(sytPerm T)` is between `of(p)` and `a_λ`, so we can't apply left absorption directly. We'd need `of(p) * (of(sytPerm T) * a_λ) = of(sytPerm T) * a_λ`... which is `of(p) * x = x` only if `x` is invariant under left P_λ-multiplication. `of(sytPerm T) * a_λ` is NOT invariant under left P_λ-multiplication in general.

This confirms `column_standard_in_span'` genuinely needs a new approach.
