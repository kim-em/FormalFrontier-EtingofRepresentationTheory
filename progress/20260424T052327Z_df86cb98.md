## Accomplished

Closed feature issue #2498 (Wall 3 #2450 sibling B): added the
**pigeonhole collapse lemma** `garnir_pigeonhole_collapse` to
`EtingofRepresentationTheory/Chapter5/SpechtModuleBasis.lean`
(insertion before `garnir_twisted_in_lower_span`, lines 978–1064 of
the new file).

**Statement (Form 1, direct).** Given `w σ : Equiv.Perm (Fin n)`, an
element `t₀ ∈ ColumnSubgroup n la` with `sign(t₀) = -1`, and a
constancy hypothesis
`∀ q ∈ ColumnSubgroup n la, toTabloid (w · q⁻¹ · σ) = toTabloid (w · σ)`,
the alternating sum
`Σ_{q ∈ Q_λ} sign(q) · [w · q⁻¹ · σ]` vanishes in `M^λ`.

**Why Form 1 over Form 2.** The structural Form 2 (existence of an
overflowing row) implies Form 1 and is the natural way callers will
discharge `hcollapse`, but encoding the overflow condition into the
lemma signature would over-specialize. The pigeonhole-on-rows
combinatorics live in the caller (sibling C), which can pick the
overflow witness based on `(w, σ, p₁, p₂)`. The collapse lemma itself
is a clean algebraic identity.

**Proof structure (≈90 lines including doc).**
1. `Σ_{q ∈ Q_λ} sign(q) = 0` via the involution `q ↦ t₀ · q`. The
   involution flips sign by `sign(t₀) = -1`, so the sum equals its own
   negation; hence `2 · sum = 0`, so `sum = 0` (since `(2 : ℂ) ≠ 0`).
   Mirrors the `garnirAnnihilate_tabloid` involution pattern but on
   `ColumnSubgroup` rather than the supported-perms type.
2. `hcollapse` rewrites each `[w · q⁻¹ · σ]` to the constant `[w · σ]`.
3. Pull the constant out via `← Finset.sum_smul`, multiply by 0, done.

**Validation on the counter-example.** Already done in
`progress/20260424T034536Z_2a4d9cc5.md`: for λ=(2,2), σ=swap(0,1), the
products `w · q⁻¹ · σ` (q ∈ Q_(2,2)) all yield `[{1,2}|{0,3}]` for
`w ∈ {swap(1,2), (0,1,2)}`. The constancy hypothesis is satisfied;
Q_(2,2) contains the column transposition `swap(0,2)` (sign = -1);
hence `garnir_pigeonhole_collapse` applies and the sum vanishes —
matching the hand-computed `TP(swap(1,2), σ) = 0` and
`TP((0,1,2), σ) = 0`.
