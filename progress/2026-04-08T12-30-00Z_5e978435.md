### Root cause analysis

The synthesis error "synthesized type class instance is not definitionally equal to
expression inferred by typing rules" was caused by:

1. `attribute [instance] QuiverRepresentation.instModule` makes *all* reps' Module instances
   globally discoverable
2. When `fp = reflectionFunctorPlus Q' i ρ'` is in scope, `fp.obj v` for `v ≠ i` reduces to
   `ρ'.obj v` (same carrier type)
3. Lean finds TWO Module instances for this type: `fp.instModule v` (from quiver
   `reversedAtVertex Q' i`) and `ρ'.instModule v` (from quiver `Q'`)
4. These are not definitionally equal (different quiver structures), triggering the error

### Solution

- Changed `set Q'` to `let Q'` (reduces instance pollution)
- Added `letI : Quiver (Fin n) := Q` to make Q the primary synthesized instance
- Removed outer `haveI` for Free/Finite of ρ' (was the direct trigger of the conflict)
- Moved `haveI` for Free/Finite into inner proof scopes where needed
- Used `hinvol ▸ fp` transport pattern: define `ρ_result : @QuiverRepresentation k (Fin n) _ Q := hinvol ▸ fp`
  and sorry the property proofs separately
- Passed `hfree'`/`hfinite'` explicitly to `@Proposition6_6_5_sink` and
  `@reflFunctorPlus_free_eq`/`ne` calls instead of relying on synthesis
