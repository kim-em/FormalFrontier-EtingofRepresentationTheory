## Current state

`polynomialRep_embeds_in_tensorPower` at
`EtingofRepresentationTheory/Chapter5/PolynomialRepEmbedding.lean:474`
(merged via PR #2539 as part of Schur-Weyl #2b equivariance) currently
bundles three matrix-coefficient hypotheses into its `hpoly` existential:

1. `hhom` — homogeneity of the polynomials `P a c` (degree `n`).
2. `hP`   — `b.repr (ρ g (b c)) a = eval (g_ij) (P a c)` for all `g`.
3. `hP_mul` — the **polynomial-level** matrix-coefficient
   multiplicativity
   `polyRightTransl g (P a c') = ∑ c, eval (g_ij) (P c c') • P a c`.

The last one, `hP_mul`, holds **at the level of evaluations** for every
`g ∈ GL_N` — it is a direct consequence of `ρ.map_mul`, the
matrix-coefficient definition, and the right-translation identity. The
polynomial-level statement requires lifting an evaluation identity to a
polynomial identity, which is available over a field of characteristic
zero (hence `Infinite k`) by `MvPolynomial.funext` + the
determinant/generic-point trick: the set of invertible matrices is
Zariski-dense in `Matrix (Fin N) (Fin N) k` as `k`-points because
`detPoly ≠ 0` in `MvPolynomial`.

Exposing `hP_mul` as a caller-provided hypothesis was a deliberate
deferral in #2539 (see the theorem docstring in PolynomialRepEmbedding.lean
lines 456-473). It keeps the equivariance assembly focused. But every
downstream consumer (notably Schur-Weyl #5 = #2482) will need to discharge
this hypothesis to cite the theorem, so we should derive it once, here,
as a shared lemma.

See the progress note `progress/20260424T083321Z_271cd1ab.md` §"Next step"

### (a) The polynomial-identity-from-evaluation lemma (core)

A `MvPolynomial`-level lemma over a field `k` with `Infinite k`
(derivable from `CharZero k`) of approximately the form:

```lean
/-- Two polynomials in `MvPolynomial (Fin N × Fin N) k` are equal if
they agree when evaluated at every invertible matrix, assuming `Infinite k`.
Proof via the determinant polynomial (generic invertibility) and
`MvPolynomial.funext`. -/
lemma MvPolynomial.eq_of_eval_eq_on_gl
    {k : Type*} [Field k] [Infinite k] {N : ℕ}
    (p q : MvPolynomial (Fin N × Fin N) k)
    (h : ∀ g : Matrix.GeneralLinearGroup (Fin N) k,
           MvPolynomial.eval
             (fun ij : Fin N × Fin N => (g : Matrix (Fin N) (Fin N) k) ij.1 ij.2) p =
           MvPolynomial.eval
             (fun ij : Fin N × Fin N => (g : Matrix (Fin N) (Fin N) k) ij.1 ij.2) q) :
    p = q
```

**Proof outline**:
1. By `MvPolynomial.funext` over an infinite field, it suffices to show
   the two polynomials agree at **every** `Fin N × Fin N → k`, not just
   invertible ones.
2. Consider the polynomial `r := (p - q) * detPoly` where
   `detPoly := MvPolynomial.det (fun i j => MvPolynomial.X (i, j))` is
   the generic determinant (a `MvPolynomial (Fin N × Fin N) k`).
3. For every `M : Fin N × Fin N → k`:
   - If `M` is invertible (`det M ≠ 0`), eval `r` at `M` is
     `(eval_M p - eval_M q) * det M = 0` by hypothesis (the invertible
     matrix is a `GL_N`-element).
   - If `M` is singular (`det M = 0`), eval `r` at `M` is `0` directly.
4. Hence `r = 0` by `MvPolynomial.funext` on `k^{N×N} → k` (infinite
   domain, and both sides of the evaluation agree).
5. `r = (p - q) * detPoly = 0` and `detPoly ≠ 0` (it's a nonzero
   polynomial over a field, hence not a zero-divisor in `MvPolynomial`),
   so `p - q = 0`, i.e. `p = q`.

### (b) The derivation lemma

```lean
/-- Derive `hP_mul` from `ρ.map_mul` and the matrix-coefficient setup.
Given the hypotheses `hhom` (homogeneity) and `hP` (matrix-coefficient
evaluation), the polynomial-level identity
`polyRightTransl g (P a c') = ∑ c, eval (g_ij) (P c c') • P a c`
holds as polynomials in `MvPolynomial (Fin N × Fin N) k`. -/
lemma hP_mul_of_hP
    {k : Type*} [Field k] [CharZero k] [IsAlgClosed k]
    {M : Type*} [AddCommGroup M] [Module k M] [Module.Finite k M]
    {N n d : ℕ}
    (b : Module.Basis (Fin d) k M)
    (P : Fin d → Fin d → MvPolynomial (Fin N × Fin N) k)
    (ρ : Matrix.GeneralLinearGroup (Fin N) k →* (M →ₗ[k] M))
    (hhom : ∀ a c, (P a c).IsHomogeneous n)
    (hP : ∀ (g : Matrix.GeneralLinearGroup (Fin N) k) a c,
           b.repr (ρ g (b c)) a =
             MvPolynomial.eval
               (fun ij : Fin N × Fin N =>
                 (g : Matrix (Fin N) (Fin N) k) ij.1 ij.2)
               (P a c))
    (g : Matrix.GeneralLinearGroup (Fin N) k) (a c' : Fin d) :
    PolynomialTensorBridge.polyRightTransl k N
        (g : Matrix (Fin N) (Fin N) k) (P a c') =
      ∑ c, MvPolynomial.eval
             (fun ij : Fin N × Fin N =>
               (g : Matrix (Fin N) (Fin N) k) ij.1 ij.2)
             (P c c') • P a c
```

Actually the parametrisation is probably cleaner as a polynomial-only
statement: re-express the target as the equality of two polynomials
(not parameterised by `g`) derived from `ρ`'s multiplicativity evaluated
symbolically. The exact phrasing is at the author's discretion — what
matters is that this replaces the user-supplied `hP_mul` in any downstream
call to `polynomialRep_embeds_in_tensorPower`.
