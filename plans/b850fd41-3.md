## Why this audit matters

PR #2665 is the abstract algebraic anchor for the entire C-4a-i
decomposition (#2655 sub-α / #2656 sub-β / #2657 sub-γ). If
`youngSym_block_factorization`'s statement is subtly wrong,

assembly consume — a defect in the refactor (e.g., a helper that's
not definitionally equal to the inline form) is the kind of thing
that propagates as opaque "term has type X but is expected to have
type Y" errors at consumer sites and wastes downstream worker time.
